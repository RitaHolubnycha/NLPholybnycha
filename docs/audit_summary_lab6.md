# Audit Summary — Lab 6 (TF-IDF + Logistic Regression baseline)

## 1. Підзадача
Класифікація токенів за частинами мови (UPOS) на основі датасету ParlaMint-UA.

## 2. Використаний split
Використано готовий split з датасету (train / dev / test), зафіксований у Lab 5.

## 3. Baseline 1
- Текст: token
- TF-IDF: word unigrams (1,1)
- Модель: Logistic Regression

**Результати:**
- Accuracy: 0.7708
- Macro-F1: 0.6631

## 4. Baseline 2
- Текст: token + lemma
- TF-IDF: word n-grams (1,2), sublinear_tf=True
- Модель: Logistic Regression

**Результати:**
- Accuracy: 0.8216
- Macro-F1: 0.6947

## 5. Порівняння
Baseline 2 показав кращі результати за рахунок:
- використання lemma
- використання біграм
- більш інформативного представлення тексту

## 6. Основні помилки
Найчастіші типи помилок:
- класифікація слів як PUNCT (over-prediction)
- короткі токени без контексту ("і", "в", "я")
- числові та спеціальні токени ("116", "41-а")
- невідомі або іншомовні токени (AliExpress, Play)

## 7. Висновки
- модель сильно залежить від форми токена
- відсутність контексту є головним обмеженням
- TF-IDF baseline працює добре як стартове рішення

## 8. Що покращити далі
- додати контекст (вікно слів)
- обробка чисел і спецтокенів
- балансування класів
- використання більш складних моделей (BiLSTM / Transformer)
