# Лабораторна робота 6 — TF-IDF + Logistic Regression baseline

## Напрям
A — класифікація (визначення частин мови / POS tagging на рівні токенів)

## Постановка задачі
Задача — класифікувати кожен токен (слово) у тексті за його частиною мови (upos).
Модель працює на рівні окремих токенів без використання повного контексту речення.

## Використані дані
- Дані з попередніх лабораторних (processed_v2)
- Готовий split з ЛР5:
  - train
  - validation
  - test

## Baseline-моделі

### Baseline 1
- Вхід: тільки token
- TF-IDF: word (1,1)
- Модель: Logistic Regression

### Baseline 2
- Вхід: token + lemma
- TF-IDF: word (1,2), sublinear_tf=True
- Модель: Logistic Regression

## Результати

Baseline 1:
- Accuracy: 0.77
- Macro-F1: 0.66

Baseline 2:
- Accuracy: 0.82
- Macro-F1: 0.69

Baseline 2 показав кращий результат завдяки:
- використанню леми
- використанню біграм

## Основні проблеми моделі

Під час аналізу помилок виявлено:

- модель часто класифікує токени як PUNCT
- короткі слова (наприклад: "і", "в", "я") часто визначаються неправильно
- числові токени (наприклад: "116") також часто класифікуються як PUNCT
- рідкісні або специфічні токени (AliExpress, Play) класифікуються некоректно

## Причини помилок

- модель працює на рівні окремих токенів без контексту
- короткі токени не дають достатньо інформації для TF-IDF
- дисбаланс класів (PUNCT зустрічається дуже часто)

## Подальші покращення

- використання char n-grams
- додавання контексту (сусідні токени)
- балансування класів
- використання більш складних моделей
